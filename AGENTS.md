# AGENTS.md

## 项目约束

### 技术栈
- 后端必须使用 Go 1.22+

### 安全约束
- 禁止在代码中硬编码密钥，必须使用环境变量
- 所有用户输入必须经过 validate 包校验

### 测试约束
- 所有新增代码必须附带单元测试（覆盖率 > 80%）
- 禁止在测试中使用真实外部服务

### Git 约束
- 提交信息必须遵循 Conventional Commits
- 禁止直接提交到 main 分支，必须通过 PR
- PR 必须通过 CI 检查才能合并

### 其他规范
- 做好版本规范和管理
- 通过progress.md记录工作进度和任务状态
- 每个工作完成后要自我验证通过后才允许继续

## 强制约束（MUST）

## 禁止事项（MUST NOT）

1. 禁止在代码中硬编码任何密钥、密码、Token
2. 禁止 handler 层直接调用 repository
3. 禁止在业务代码中使用 `fmt.Println`，必须使用结构化日志（zap）
4. 禁止提交 `go.sum` 以外的不必要文件
